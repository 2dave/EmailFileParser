Hi all,<br>
<br>
I've run into an interesting issue defining directory structure for
merge modules. I've solved it but I feel it is a bit of a kludge; see
what you think.<br>
<br>
The idea is to use merge configuration to allow a target data folder to
be re-defined when my merge module is integrated into an msi.<br>
I have registry values pointing to three folders and I also create the
folders. This requires the folders to be defined in the directory table.<br>
<br>
My definition for the folders is:<br>
<br>
&lt;Directory Id=&quot;Dummy&quot; Name=&quot;SourceDir&quot;&gt;<br>
&nbsp; &lt;Directory Id=&quot;CommonAppDataFolder&quot;&gt;<br>
&nbsp;&nbsp;&nbsp; &lt;Directory Id=&quot;CompanyDataRoot&quot; Name=&quot;MyCompany&quot;&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;Directory Id=&quot;AOService_DataFolderRoot&quot; Name=&quot;Services&quot;&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;Directory
Id=&quot;AODataFolderRoot&quot; Name=&quot;AOutput&quot; LongName=&quot;AutoOutput&quot;&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;Directory Id=&quot;AODataFolder&quot; Name=&quot;.&quot;/&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
&lt;Directory Id=&quot;AOTempDataRoot&quot; Name=&quot;AOTemp&quot;&gt;<br>
&nbsp;&nbsp;
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
&lt;Directory Id=&quot;AOTempData&quot; Name=&quot;.&quot;/&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/Directory&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
&lt;Directory Id=&quot;AOExportDataRoot&quot; Name=&quot;AOExport&quot;&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
&lt;Directory Id=&quot;AOExportData&quot; Name=&quot;.&quot; /&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/Directory&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/Directory&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/Directory&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/Directory&gt;<br>
&nbsp;&nbsp;&nbsp;&nbsp; &lt;/Directory&gt;<br>
&nbsp; &lt;/Directory&gt;<br>
<br>
Notice I've used the CommonAppDataFolder with the aim of placing this (shared) data in<br>
&nbsp;&quot;..\Documents and Settings\All Users\Application Data\My Company\...&quot;<br>
<br>
Problem 1: CommonAppDataFolder is modularized when the msm is built, thus having a guid appended and not pointing where I want.<br>
Solution 1: In the merge substitution table I added this row:<br>
<br>
&lt;Substitution Table=&quot;Directory&quot; Row=&quot;CompanyDataRoot&quot; Column=&quot;Directory_Parent&quot; Value=&quot;CommonAppDataFolder&quot;/&gt;<br>
<br>
Is this a good idea? Of course, it breaks if the msm is not used through mergemod.dll.<br>
<br>
Problem 2: The default value for merge configuration data does not account for modularised (added GUID) directory ids.<br>
Therefore when I attempt to defing a default location for the folders
for anyone not providing them, the wrong value is used in the
substitution table.<br>
Solution 2: I manually added the GUID. Maybe this would break in a future WiX implementation?<br>
<br>
Eg.<br>
&lt;Configuration Name=&quot;AOTempPath&quot; Format=&quot;Key&quot; Type=&quot;Property&quot;
DisplayName=&quot;AOTempPath&quot; ContextData=&quot;DataPaths&quot;
DefaultValue=&quot;AODataFolderRoot.7616849F_04FA_41AF_89C8_0F3D48CDA145&quot;/&gt;<br>
<br>
which is used as:<br>
<br>
&lt;Substitution Table=&quot;Directory&quot; Row=&quot;AOTempDataRoot&quot;
Column=&quot;Directory_Parent&quot;
Value=&quot;[=AOTempPath]&quot;/&gt;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <br>
<br>
Comments and ideas welcome.<br>
